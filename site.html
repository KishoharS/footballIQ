How it works

The RAG pipeline.

Five stages, from raw CSV to grounded natural language answers.

01

📄

Ingest

EA FC 26 player CSVs loaded with LangChain's CSVLoader. 16,254 player profiles and goalkeeper records parsed into Document objects.

CSVLoader

02

✂️

Chunk

RecursiveCharacterTextSplitter breaks documents into 1000-character chunks with 100-character overlap to preserve context at boundaries.

chunk_size=1000

03

🔢

Embed

all-MiniLM-L6-v2 from sentence-transformers encodes each chunk into a 384-dimensional vector capturing semantic meaning.

384-dim vectors

04

🗄️

Store

All vectors persisted in ChromaDB's local vector store. PersistentClient ensures data survives kernel restarts without re-embedding.

PersistentClient

05

🤖

Generate

A LangGraph ReAct agent autonomously decides when to query the vector store, retrieves top-k chunks, and grounds the LLM's answer in real data.

ReAct · Groq

Endpoints

Simple. Restful. Ready.

Two endpoints. One for answers, one for health checks.

POST

/ask

Send a natural language question, receive a grounded answer retrieved from the EA FC vector store.

# Request { "query": "What is Salah's pace?" } # Response { "response": "Salah's PAC is 89..." }
GET

/health

Verify the service is running. Returns a simple status object for uptime monitoring and deployment checks.

# Response { "status": "ok" } # Run locally uvicorn api:app --reload
